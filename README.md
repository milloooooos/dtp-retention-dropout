# DTP 患者服务 · 留存率 / 脱落率 自助分析

上传 **销售底表** 与 **随访任务表**，自动计算 DTP 药房患者管理的核心留存/脱落指标，并把"销售算出的脱落患者"与"随访记录的脱落原因"跨表关联，输出可控/不可控构成与行动建议。

> 面向 AZ（阿斯利康）等药企委托 DTP 药房的患者服务复盘场景。数据全程在本地/浏览器处理，不上传第三方。

## 功能

| 模块 | 口径 | 说明 |
|------|------|------|
| **留存率 Cohort** | 按首购月分群（=新患队列） | Mk 留存% = 该群在首购后第 k 月「有购药」的患者占比 |
| **脱落率 A（滚动）** | 基准月 M-2 有购药、M-1∪M 无购药 → 脱落 | 月度口径，适合看趋势 |
| **脱落率 B（累计沉默）** | 末次购药距数据终点 > N×用药间隔 → 真停药 | 另给「已观察%」剔除新近患者右删失 |
| **跨表关联** | 销售脱落患者 ↔ 随访脱落原因 | 品牌层面关联（可靠）+ 近似患者级（线索） |
| **脱落映射** | 原因文本 → 可控/不可控 | 关键词分类 + 行动建议映射 |

## 快速开始（本地）

```bash
pip install -r requirements.txt
streamlit run app.py
# 浏览器打开 http://localhost:8501
```

1. 上传「销售底表」（xlsx/xls）
2. 可选上传「随访任务表」（可多选 2025H1 + 2026H1 两个 xls）
3. 调侧边栏参数（留存追踪月数、脱落率B倍数、是否输出近似患者级）
4. 点「运行分析」→ 看图表/表格 → 点「下载完整 Excel 报告」

> 不传随访表也能算留存率与脱落率，只是没有跨表脱落原因。

## 数据要求

**销售底表**（Sheet 含以下列，列名可用等价别名，引擎自动识别）：
`商品名称`（含品牌括号，如"甲磺酸奥希替尼片(泰瑞沙)"）、`销售时间`、`销售数量`、`患者ID`（或会员号/开票抬头）、`会员姓名`（可选）、`药房名称`、`医疗单位`、`处方医生`。

**随访任务表**（xls，历史任务导出）：
`药品名称`、`患者`（姓名）、`用药周期状态`、`未按计划持续用药原因`、`本月未购药的原因`、`患者首购日期`、`末次购药日期`、`创建日期`、`任务类型`。

> 支持品牌：泰瑞沙 / 优赫得 / 利普卓 / 英飞凡 / 沃瑞沙 / 凡舒卓 / 荃科得。用药间隔（每盒天数）内置，可在 `dtp_engine.DAYS_PER_BOX` 调整。

## 重要口径边界

- **留存率可非单调递减**：因患者一次购多盒，次月不买、第 3 月才回，属真实数据特征；近期 cohort 窗口不足显示为空（右删失）。
- **两表无共同患者主键**：销售表会员号（浮点格式）与随访表患者编码/会员号（字符串）格式不同，无法直接对患者关联。故：
  - **品牌层面关联为主**（可靠）：各品种销售脱落率 ↔ 各品种随访原因可控占比。
  - **近似患者级关联**（置信度低）：按 姓名+品种+首购月 匹配，同名歧义大，仅作线索排查。
- **随访覆盖缺口**：若随访表某品种为 0 行（如本例优赫得），跨表该品种原因为空，属覆盖缺口非计算错误。
- **脱落率 B 右删失**：新近患者（首购在终点前 N×间隔内）未到判断停药的时机，已用「已观察%」剔除。
